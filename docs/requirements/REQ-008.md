# Description

The E2E tests must be integrated into the CI/CD pipeline. Specifically, tests must run on every push to the main branch using GitHub Actions. Additionally, the project README must display a badge indicating the current test status.

# Acceptance Criteria

- [ ] A GitHub Actions workflow is configured to run E2E tests.
- [ ] The workflow triggers on every push to the "main" branch.
- [ ] The README.md file includes a badge showing the current status of the test run.
