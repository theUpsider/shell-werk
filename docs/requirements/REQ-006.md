# Description

The application shall provide an "Auto Execute" mode for shell commands within a single chat session. This allows the user to bypass the confirmation step for subsequent shell commands in that session.

# Acceptance Criteria

- [ ] The user can enable "Auto Execute" mode for the current chat session.
- [ ] The "Auto Execute" option is available as a checkbox under the first tool call in a session.
- [ ] When "Auto Execute" is enabled, shell commands are executed immediately without asking for user confirmation.
- [ ] The "Auto Execute" setting applies only to the current chat session.
- [ ] If Chat Only mode is enabled, Auto Execute is ignored and no commands run.
